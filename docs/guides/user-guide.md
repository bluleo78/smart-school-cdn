## 1. 서비스 개요

### 이 서비스가 하는 일

학교 내부에 캐시 서버를 두어 디지털 교과서의 이미지·영상 등 정적 콘텐츠를 로컬에서 빠르게 제공합니다.

```
[Before]  iPad → 인터넷 → cdn.textbook.com → 이미지 다운로드 (느림)
[After]   iPad → 학교 CDN 서버 → 로컬 캐시에서 즉시 응답 (빠름)
```

### 핵심 원리

- 교과서 **콘텐츠 CDN 도메인만** 학교 서버로 우회 (DNS 오버라이드)
- 인증/로그인 서버는 **그대로 원본**으로 통과 — 비밀번호가 CDN을 경유하지 않음
- 앱 수정 불필요 — 네트워크 레벨에서 투명하게 동작

### 보안 경계

| 구분 | 도메인 예시 | 처리 방식 |
|------|------------|----------|
| **프록시 대상** | `cdn.textbook.com` | CDN 서버가 캐싱 후 응답 |
| **바이패스** | `auth.textbook.com`, `api.textbook.com`, `login.textbook.com` | 원본 서버로 직접 통과 |
| **기타 모든 도메인** | `google.com`, `youtube.com` 등 | 원본 서버로 직접 통과 |

## 2. 전체 동작 흐름

### 2-1. 큰 그림

```
┌──────────────────────────────────────────────────────────────────┐
│                        학교 내부 네트워크                          │
│                                                                  │
│  ┌─────────┐                                                     │
│  │  iPad   │                                                     │
│  │ 교과서앱 │                                                     │
│  └────┬────┘                                                     │
│       │                                                          │
│       │ ① DNS 질의: cdn.textbook.com 의 IP는?                    │
│       ▼                                                          │
│  ┌──────────┐                                                    │
│  │ CDN DNS  │  cdn.textbook.com → 10.0.1.100 (CDN 서버)          │
│  │ (53)     │  auth.textbook.com → 원본 IP (그대로 통과)           │
│  └──────────┘                                                    │
│       │                                                          │
│       │ ② HTTPS 요청: GET https://cdn.textbook.com/img/p32.jpg   │
│       ▼                                                          │
│  ┌───────────┐    캐시 HIT     ┌───────────┐                     │
│  │ CDN Proxy │ ──────────────→ │ Storage   │ → 즉시 응답          │
│  │ (443)     │                 └───────────┘                     │
│  │           │    캐시 MISS                                      │
│  │           │ ──→ 원본 서버에서 다운로드                           │
│  │           │ ──→ (선택) Optimizer로 이미지 최적화                 │
│  │           │ ──→ Storage에 저장                                 │
│  │           │ ──→ iPad에 응답                                    │
│  └───────────┘                                                   │
│                                                                  │
│  ┌───────────┐                                                   │
│  │ Admin     │  관리자 대시보드 (http://10.0.1.100:3000)           │
│  │ Dashboard │  도메인 관리, 캐시 퍼지, 통계 확인                    │
│  └───────────┘                                                   │
└──────────────────────────────────────────────────────────────────┘
              │
         [외부 인터넷]
         cdn.textbook.com (원본 서버)
```

### 2-2. 첫 번째 학생이 접속할 때 (콜드 캐시)

```
학생 A의 iPad                CDN 서버                    원본 서버
    │                          │                           │
    │  DNS: cdn.textbook.com?  │                           │
    │ ────────────────────────→│                           │
    │  응답: 10.0.1.100        │                           │
    │ ←────────────────────────│                           │
    │                          │                           │
    │  GET /chapter1/img1.jpg  │                           │
    │ ────────────────────────→│                           │
    │                          │  캐시 MISS                │
    │                          │  GET /chapter1/img1.jpg   │
    │                          │ ─────────────────────────→│
    │                          │  200 OK (2MB PNG)         │
    │                          │ ←─────────────────────────│
    │                          │                           │
    │                          │  [최적화: PNG→WebP 200KB] │
    │                          │  [캐시에 저장]             │
    │                          │                           │
    │  200 OK (200KB WebP)     │                           │
    │ ←────────────────────────│                           │
```

### 2-3. 같은 반 다른 학생이 접속할 때 (캐시 히트)

```
학생 B의 iPad                CDN 서버
    │                          │
    │  GET /chapter1/img1.jpg  │
    │ ────────────────────────→│
    │                          │  캐시 HIT (로컬 디스크)
    │  200 OK (200KB WebP)     │  ← 원본 서버 요청 없음!
    │ ←────────────────────────│
    │                          │
    ※ 응답 시간: 수 ms (vs 원본 수백 ms~수 초)
```

### 2-4. 인증 요청은 CDN을 경유하지 않음

```
학생의 iPad                  CDN DNS                    인증 서버
    │                          │                           │
    │  DNS: auth.textbook.com? │                           │
    │ ────────────────────────→│                           │
    │  응답: 203.0.113.50      │  ← 원본 IP 그대로 반환    │
    │ ←────────────────────────│                           │
    │                          │                           │
    │  POST /login (ID/PW)     │                           │
    │ ────────────────────────────────────────────────────→│
    │  200 OK (토큰 발급)       │                           │
    │ ←────────────────────────────────────────────────────│
    │                          │                           │
    ※ CDN 서버를 경유하지 않으므로 비밀번호 노출 없음
```

## 3. CDN 서버 설치

### 3-1. 요구 사항

| 항목 | 최소 사양 | 권장 사양 |
|------|----------|----------|
| OS | Ubuntu 22.04+ / macOS | Ubuntu 24.04 LTS |
| CPU | 2코어 | 4코어 |
| RAM | 4GB | 8GB |
| 디스크 | 50GB SSD | 200GB+ SSD |
| 네트워크 | 고정 IP (예: 10.0.1.100) | 기가비트 |
| 소프트웨어 | Docker, Docker Compose | |

### 3-2. 설치

```bash
# 1. 소스 다운로드
git clone https://github.com/your-org/smart-school-cdn.git
cd smart-school-cdn

# 2. 환경 변수 설정
cp .env.example .env
```

`.env` 파일을 편집합니다:

```env
# CDN 서버의 내부 IP (태블릿이 접속할 주소)
CACHE_SERVER_IP=10.0.1.100

# 외부 DNS (캐싱 대상이 아닌 도메인 해석용)
UPSTREAM_DNS=8.8.8.8

# 캐시 최대 용량
CACHE_MAX_SIZE=50g

# CA 인증서 이름 (태블릿에 표시될 이름)
CA_SUBJECT=OO학교 CDN CA
```

```bash
# 3. 서비스 시작
docker compose -f docker-compose.prod.yml up -d

# 4. 상태 확인
docker compose -f docker-compose.prod.yml ps
```

### 3-3. 서비스 확인

```bash
# Admin Dashboard 접속 확인
curl http://10.0.1.100:3000

# DNS 서비스 확인
dig @10.0.1.100 google.com  # 외부 도메인 → 정상 해석되어야 함
```

## 4. 초기 설정 (Admin Dashboard)

브라우저에서 `http://10.0.1.100:3000` 접속합니다.

### 4-1. 캐싱 대상 도메인 등록

**도메인 관리** 메뉴에서 교과서 콘텐츠 CDN 도메인을 추가합니다.

| 설정 | 값 | 설명 |
|------|-----|------|
| 도메인 | `cdn.textbook.com` | 교과서 앱이 콘텐츠를 가져오는 도메인 |
| 유형 | 정적 콘텐츠 | 이미지, 영상, CSS, JS 등 |
| 캐시 TTL | 7일 | 콘텐츠 갱신 주기에 맞게 조정 |

> **주의**: `auth.textbook.com`, `api.textbook.com` 등 인증/API 도메인은 **등록하지 않습니다**.

도메인을 등록하면 자동으로:
1. TLS Service가 해당 도메인의 인증서를 발급합니다
2. DNS Service가 해당 도메인을 CDN 서버 IP로 오버라이드합니다

### 4-2. 최적화 설정 (선택)

**최적화 설정** 메뉴에서 이미지 최적화 프로파일을 설정합니다.

| 설정 | 권장값 | 설명 |
|------|--------|------|
| 이미지 포맷 | WebP | 원본 대비 60~80% 용량 절감 |
| 품질 | 80% | 시각적 차이 거의 없음 |
| 최대 해상도 | 2048px | 태블릿 화면 크기에 맞춤 |

### 4-3. CA 인증서 다운로드

**시스템** 메뉴 → **CA 인증서 다운로드** 버튼으로 인증서 파일을 받습니다.

- `smart-school-cdn-ca.crt` — 태블릿에 설치할 CA 인증서
- `smart-school-cdn-ca.mobileconfig` — iPad용 프로파일 (iOS 설정 간편화)

## 5. 네트워크 설정 (DNS)

태블릿이 CDN 서버의 DNS를 사용하도록 설정합니다. 방법은 2가지입니다.

### 방법 A: 학교 공유기/DHCP에서 DNS 변경 (권장)

공유기 관리 페이지에서 DHCP가 배포하는 DNS 서버를 CDN 서버 IP로 변경합니다.

```
기존:  DNS → 8.8.8.8  또는 통신사 DNS
변경:  DNS → 10.0.1.100 (CDN 서버)
```

이렇게 하면 학교 Wi-Fi에 연결된 **모든 기기가 자동으로** CDN DNS를 사용합니다.
캐싱 대상이 아닌 도메인은 CDN DNS가 외부 DNS로 포워딩하므로 일반 인터넷은 정상 동작합니다.

### 방법 B: MDM으로 태블릿 DNS 개별 설정

DHCP 변경이 불가능한 경우, MDM에서 Wi-Fi 프로파일의 DNS를 CDN 서버 IP로 지정합니다.

## 6. 태블릿 CA 인증서 설치

CDN 서버가 `cdn.textbook.com`을 대신 응답하려면 자체 TLS 인증서를 사용합니다.
태블릿이 이 인증서를 신뢰하도록 **CA 인증서를 설치**해야 합니다.

### 방법 A: MDM 일괄 배포 (권장)

MDM(Jamf, Intune, Knox 등)에서 CA 인증서를 포함한 프로파일을 배포합니다.

```
MDM 관리 콘솔
  → 프로파일 생성
  → 인증서 페이로드 추가
  → smart-school-cdn-ca.crt 업로드
  → 대상 기기 그룹에 배포
```

수백 대 태블릿에 자동으로 설치되며, 관리자가 원격으로 제거할 수도 있습니다.

### 방법 B: iPad 수동 설치

소수의 기기이거나 MDM이 없는 경우:

1. CA 인증서 파일을 iPad로 전송합니다
   - AirDrop, 이메일 첨부, 또는 `http://10.0.1.100:3000/ca.crt` 다운로드

2. 파일을 열면 **"프로파일이 다운로드됨"** 알림이 표시됩니다

3. `설정` → `일반` → `VPN 및 기기 관리` → 다운로드된 프로파일 선택 → **설치**

4. `설정` → `일반` → `정보` → `인증서 신뢰 설정` → 해당 CA **신뢰 토글 활성화**

> ⚠ **4번을 빠뜨리면** 인증서가 설치되어 있지만 신뢰되지 않아 앱에서 여전히 오류가 발생합니다.

### 방법 C: Galaxy Tab 수동 설치

1. CA 인증서 파일을 태블릿으로 전송합니다

2. `설정` → `생체 인식 및 보안` → `기타 보안 설정` → `인증서 설치`
   (또는 `설정` → `보안` → `인증서 설치`)

3. 파일 선택 → 인증서 이름 입력 → **Wi-Fi** 용도 선택 → 설치

## 7. 동작 확인

### 7-1. DNS 확인

태블릿의 브라우저에서 확인하거나, CDN 서버에서 테스트합니다:

```bash
# CDN 서버에서 테스트
dig @10.0.1.100 cdn.textbook.com
# 응답이 10.0.1.100 이면 정상

dig @10.0.1.100 auth.textbook.com
# 응답이 원본 서버 IP이면 정상 (CDN IP가 아니어야 함)
```

### 7-2. 캐싱 확인

```bash
# 첫 요청 (MISS)
curl -v https://cdn.textbook.com/test/image.jpg --resolve cdn.textbook.com:443:10.0.1.100
# 응답 헤더: X-Cache-Status: MISS

# 두 번째 요청 (HIT)
curl -v https://cdn.textbook.com/test/image.jpg --resolve cdn.textbook.com:443:10.0.1.100
# 응답 헤더: X-Cache-Status: HIT
```

### 7-3. 태블릿에서 확인

1. 태블릿을 학교 Wi-Fi에 연결합니다
2. 교과서 앱을 실행하고 로그인합니다
3. 교과서 콘텐츠(이미지, 영상)가 정상 로딩되는지 확인합니다
4. Admin Dashboard(`http://10.0.1.100:3000`)에서 트래픽과 캐시 히트가 표시되는지 확인합니다

## 8. 일상 운영

### 8-1. Dashboard 모니터링

`http://10.0.1.100:3000` 에서 확인할 수 있는 항목:

| 지표 | 설명 |
|------|------|
| 캐시 히트율 | 높을수록 좋음 (목표: 80% 이상) |
| 대역폭 절감 | 외부 트래픽 감소량 |
| 디스크 사용량 | 캐시 저장소 용량 |
| 인기 콘텐츠 | 자주 요청되는 리소스 목록 |
| 서비스 상태 | 각 서비스의 온라인/오프라인 상태 |

### 8-2. 캐시 퍼지 (갱신)

교과서 콘텐츠가 업데이트된 경우 오래된 캐시를 삭제합니다:

- **URL별 퍼지**: 특정 리소스만 갱신
- **도메인별 퍼지**: 특정 교과서 도메인의 모든 캐시 삭제
- **전체 퍼지**: 모든 캐시 삭제 (디스크 용량 부족 시)

### 8-3. 새 교과서 도메인 추가

새 교과서 서비스가 추가되면:

1. Admin Dashboard → 도메인 관리 → 도메인 추가
2. 인증서가 자동 발급되고, DNS 오버라이드가 자동 등록됨
3. 태블릿에 이미 CA 인증서가 설치되어 있으므로 **추가 태블릿 설정 불필요**

### 8-4. 서비스 업데이트

```bash
cd smart-school-cdn

# 최신 버전 다운로드
git pull

# 서비스 재빌드 및 재시작
docker compose -f docker-compose.prod.yml up -d --build
```

## 9. 문제 해결

### 교과서 앱에서 이미지가 로딩되지 않음

| 원인 | 확인 방법 | 해결 |
|------|----------|------|
| CA 인증서 미설치 | iPad 설정 → VPN 및 기기 관리 확인 | [6장](#6-태블릿-ca-인증서-설치) 참고 |
| CA 인증서 신뢰 미활성화 | iPad 설정 → 정보 → 인증서 신뢰 설정 | 토글 활성화 |
| DNS 미적용 | `nslookup cdn.textbook.com` 결과 확인 | CDN 서버 IP가 반환되어야 함 |
| CDN 서비스 중단 | Dashboard 접속 확인 | `docker compose ps`로 서비스 상태 확인 |

### 캐시 히트율이 낮음

| 원인 | 해결 |
|------|------|
| 서비스 시작 직후 | 시간이 지나면 자연히 올라감 |
| 캐시 용량 부족 | `.env`의 `CACHE_MAX_SIZE` 증가 |
| TTL이 너무 짧음 | 도메인 설정에서 캐시 TTL 조정 |

### CDN 서버 접속 불가

```bash
# 서비스 상태 확인
docker compose -f docker-compose.prod.yml ps

# 로그 확인
docker compose -f docker-compose.prod.yml logs --tail=50 proxy
docker compose -f docker-compose.prod.yml logs --tail=50 dns

# 서비스 재시작
docker compose -f docker-compose.prod.yml restart
```

### 특정 사이트가 접속되지 않음

CDN DNS가 외부 도메인을 정상 해석하지 못하는 경우:

```bash
# DNS 포워딩 확인
dig @10.0.1.100 google.com

# 외부 DNS 변경 (.env)
UPSTREAM_DNS=1.1.1.1
```
